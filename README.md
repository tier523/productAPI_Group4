# Projektuppgift

Skapa enhetstester (för Product.Java) och system/acceptanstester.
Tester ska skapas enligt BDD (Behavior-Driven Development). 
BDD fokuserar på beteendet av produkten från slutanvändarens perspektiv.
<br>
Skriv testfallen utifrån <i>given, when, then.</i>

## Krav
- Använda selenium
- Använda cucumber
- Enhetstester
- System/acceptanstester
- Testerna ska köras automatiskt i en CI/CD lösning
- Kommentarer ska ange vem som skrivit vilken del av koden
- Gruppdeltagarna behöver delta i gruppens agila arbete och bidra till gruppens testkod
- Dokumentation av alla sprintar
- Er sprint som SCRUM-master


## Godkänna pull request
- Sätt den som skrivit koden som asignee
- Sätt dig själv som reviewer
- Koppla pull requesten till projectet om den inte redan är det
- Koppla pull requesten till en ticket om det finns en
- Kolla så att testerna gått grönt
- Kontrollera att koden ser ok ut
- Godkän PR genom att göra 'squash & merge'
